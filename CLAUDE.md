## 코딩 핵심 원칙

- **소프트웨어의 적은 버그가 아니라 복잡성이다.** 모든 설계·구현 결정에서 복잡성을 최소화하는 선택을 우선한다. 버그는 복잡성의 증상이며, 복잡성을 줄이면 버그의 발생 여지 자체가 줄어든다. 기능 추가·추상화·의존성 도입 시 "이게 전체 복잡성을 늘리는가?"를 먼저 묻고, 늘린다면 그만한 가치가 있는지 명시적으로 정당화한다.
- 투기적 추상화 금지 — 지금 필요한 것만 구현
- 검증은 시스템 경계(사용자 입력, 외부 API)에서만
- DRY: 동일 로직 2회 이상 등장 시 즉시 함수 추출
- 함수 최대 30줄 / 파일 최대 300줄 (초과 시 분리)
- 구현 전 검증 기준(테스트 또는 성공 조건) 먼저 정의

## AI 에이전트 자율성 경계

에이전트(Claude Code)가 사람 확인 없이 어디까지 진행해도 되는지의 경계. 팀 정책에 맞게 조정하세요.

### 확인 없이 자율 실행 가능

- 파일 읽기, 코드 분석, 탐색
- 로컬 파일 생성/수정 (테스트, 문서)
- 테스트 실행, 린트 실행

### 반드시 사람이 확인 후 실행

- git push, PR 생성
- 외부 API 호출 (데이터 전송)
- 패키지 추가/삭제
- DB 마이그레이션 실행
- 배포 명령 실행
