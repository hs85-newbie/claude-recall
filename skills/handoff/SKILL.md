# /handoff — 다음 세션용 인계 메모 작성

context-save/restore의 경량 대체. **저장만 하면 다음 세션이 자동으로 읽는다**(SessionStart 훅). 사람이 restore를 기억해 호출할 필요가 없다.

**기계 자동본(SessionEnd 훅)이 이미 git 상태+마지막 지시는 남긴다.** 이 스킬은 그것만으로 부족할 때 — 결정·이유·다음 액션 같은 *모델만 아는 맥락*을 풍부하게 남길 때 쓴다.

## 절차

### 1. 저장 위치 계산
자동 로드 훅과 **동일한 키**를 써야 한다(레포 루트 기준).
```bash
ROOT=$(git rev-parse --show-toplevel 2>/dev/null || pwd)
KEY=$(printf '%s' "$ROOT" | sed 's#^/##; s#/#__#g')
DIR="$HOME/.claude/handoffs/$KEY"
TS=$(date +%Y%m%d-%H%M%S)
mkdir -p "$DIR"
echo "$DIR/$TS.md"   # 여기에 Write
```

### 2. 인계문 작성 (Write)
아래 골격으로, **이번 세션에서 실제로 일어난 것만** 적는다(지어내기 금지). 간결한 일상어 + 경로·명령어는 원문 그대로.

```markdown
# 세션 인계 메모

- 저장 시각: {TS} · 세션: {세션이름}
- 한 줄 현황: 지금 무엇을 어디까지 했나

## 하던 작업 (무엇/왜)
- 

## 확정된 결정
- 

## 다음 액션 (이어서 할 일, 우선순위 순)
1. 

## 미해결·주의
- 

## 관련 경로
- 파일/보고서/체크포인트 경로
```

### 3. 보고
채팅엔 한 줄로: 저장 경로 + "다음 세션 시작 시 자동 로드됨".

## 철칙
- 자동 로드 훅과 키 규칙(레포 루트 → `/`를 `__`로)을 어기지 말 것. 어긋나면 로드가 안 된다.
- 날짜·시각은 추정 말고 `date`로 취득.
- 영구 사실·선호(다음 세션마다 자동 적용돼야 하는 것)는 인계가 아니라 `MEMORY.md`로. 인계는 *이번 작업의 진행 상태*만.
- 외부 발행·머지 같은 컨펌 게이트 상태는 "대기 중/승인됨"을 명확히 적어 다음 세션이 멋대로 진행하지 않게.
