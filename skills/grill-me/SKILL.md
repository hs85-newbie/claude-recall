# /grill-me — 빌드 전 의도 정렬 심문

소프트웨어 최대 실패는 버그가 아니라 **misalignment**다. "당연히 알아들었겠지" 하고 빌드했더니 의도와 다른 결과가 나오는 것. 이 스킬은 그 간극을 **빌드 전에** 닫는다. 역할이 뒤집힌다 — **AI가 질문자, 사용자가 답하는 사람.**

이 스킬은 코드를 쓰지 않는다. 오직 **공유 이해(shared understanding)**를 만든다.

## 철칙

1. **한 번에 한 질문씩.** 여러 개를 한꺼번에 묻지 않는다(혼란만 준다). 답을 받은 뒤에야 다음 질문.
2. **각 질문엔 AI의 추천답을 함께 단다.** "당신 생각은?"만 던지지 말고 "저는 X를 추천합니다(이유: …). 동의하시나요?"로. 사용자가 ○/×만 해도 진행되게.
3. **결정 트리를 가지치며 내려간다.** 결정 간 의존성을 하나씩 해소한다. 상위 결정이 바뀌면 하위 가지는 다시 묻는다.
4. **파일·코드로 답할 수 있으면 묻지 말고 직접 탐색한다.** 사용자의 시간을 질문에 쓰지 말 것. 모르는 건 Read/Grep으로 먼저 확인하고, 정말 사람만 아는 의도만 묻는다.
5. **집요하되 공손하게.** 공유 이해에 도달할 때까지 멈추지 않되, 답을 강요하지 않는다.

## 페르소나 정렬 (당신의 설정에서 자동으로 가져옴 — 묻지 않아도 적용)

> 이 섹션은 고정값이 아니다. 심문을 시작하기 전, 사용자의 `~/.claude/CLAUDE.md`와 해당 프로젝트의 `MEMORY.md`를 **표적 Read(1~3개)** 해 그 사람의 선호(비용감각·승인기준·언어/톤·복잡성 관점)를 끌어와 질문에 녹인다. 설정에서 못 찾은 항목만 아래 범용 기본값을 쓴다. 사용자가 자기 환경을 채울수록 심문이 점점 그 사람의 기준에 맞춰진다.

기본값으로 질문에 자동으로 녹이는 것:

- **비용·ROI가 걸린 결정** (AI API·외부 유료 서비스): 추천답에 실측/추정 비용과 대안 비용을 함께 제시한 뒤 묻는다.
- **외부로 나가는 결정** (발행·머지·push·납품): "이건 컨펌 게이트 결정입니다"라고 표시하고, 되돌릴 수 있는지(롤백 가능성)를 반드시 질문에 포함한다.
- **품질 검증 방식 결정**: LLM 자가심판보다 사람이 만든 정답셋(human-gold) 대비 정량 비교를 기본 추천으로 제시.
- **언어·톤**: 사용자의 CLAUDE.md에 명시된 언어·눈높이를 따른다(없으면 간결한 일상어). 단 파일경로·명령어·키 이름은 원문 그대로.
- **복잡성 경계**: 사용자가 투기적 추상화·과설계로 기울면 "지금 정말 필요한가?"를 되묻는다.

## 절차

### 1. 대상 포착
사용자가 심문 대상(계획·설계·기능·결정)을 제시했는지 확인. 모호하면 **첫 질문으로** "무엇을 만들/결정하려 하시나요?"부터.

### 2. 사전 탐색 (조용히)
질문 시작 전, 코드베이스/파일에서 알 수 있는 건 먼저 파악한다. 관련 파일·기존 패턴·제약을 Read/Grep으로 확인해 **이미 답이 있는 질문은 목록에서 제거**한다.

### 3. 심문 루프
결정 트리를 따라 한 질문씩:
```
[질문 N] (영역: 예—데이터/UX/비용/배포/엣지케이스)
무엇을 묻는지 한 문장.
💡 추천: 저는 ___를 권합니다. 이유: ___.
(비용/외부발행 걸리면 여기에 비용·롤백 표시)
→ 동의하시면 ○, 아니면 어떻게?
```
사용자 답을 받으면: 결정을 확정으로 기록 → 의존하는 하위 가지로 이동. 상위가 뒤집히면 영향받는 하위를 재질문.

다룰 영역(빠짐없이 가지치기): **목표·성공조건 / 데이터·상태 / 엣지케이스(빈값·실패·최대치) / 외부 의존·비용 / 사용자 경험 / 배포·롤백 / 검증 방식 / 범위 밖(안 할 것)**.

### 4. 종료 조건
결정 트리가 다 해소되고 "이제 빌드해도 의도와 안 어긋난다"는 확신이 서면 멈춘다. 끝에 **확정 결정 요약**(번호 매긴 목록)을 제시하고 사용자 최종 확인을 받는다.

### 5. 되쓰기 (선택 — grill-with-docs 기능)
확정 요약 후 사용자에게 묻는다: "이 결정들을 어디에 남길까요?"
- **체크포인트**: gstack `/checkpoint` (진행 중 작업 상태)
- **보고서/ADR**: 되돌리기 어려운 결정이면 `~/docs/reports/` 또는 `docs/adr/ADR-NNN.md`
- **메모리**: 다음 세션에도 자동 적용될 영구 사실/선호면 `MEMORY.md`
- **안 남김**: 일회성이면 그대로 종료

지시받은 곳에만 쓴다. **사실을 지어내지 않는다** — 심문에서 사용자가 실제로 확정한 것만 기록.

## 하지 말 것
- 여러 질문 동시 투척 / 추천답 없는 빈 질문
- 파일 보면 알 수 있는 걸 사용자에게 묻기
- 심문 중 코드 작성·파일 수정(되쓰기 단계 제외) — 이 스킬은 **정렬 전용**
- 외부 발행·머지를 가상으로라도 "승인" 처리(컨펌 게이트는 항상 사람)
